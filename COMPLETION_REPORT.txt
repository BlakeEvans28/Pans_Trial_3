GAME RULES IMPLEMENTED
═══════════════════════════════════════════════════════════════════════════════

SETUP ✓
  • 6×6 grid of cards (2-10 from standard deck)
  • Each player places King at starting position (row 0 or 5, col 4)
  • 4 Jacks shuffled to determine suit roles for the round

MOVEMENT PHASE (Traversing the Labyrinth) ✓
  • 3 movement turns per player, alternating
  • 4-directional movement (up/down/left/right)
  • Toroidal wrapping at grid edges
  • Suit-based card effects:
    - Wall: Blocks movement
    - Trap: Deal damage + steal opponent's highest card
    - Ballista: Move anywhere in row/column without triggering effect
    - Weapon: Pickup for combat requirement
  • Automatic combat when players land on same cell
  • Holes created from traps/weapons

CARD PLAY PHASE (Appeasing Pan) ✓
  • Both players simultaneously play a card
  • Trump order: Walls > Traps > Ballista > Weapons
  • Tie-breaker: Highest value wins (Ace=1, Queen=11, King=12)
  • Winner chooses request first, loser second (or sits out if "Ignore Us")
  • Request options:
    - Restructure: Swap two Jacks (changes suit effects)
    - Steal Life: Swap one damage card with opponent
    - Ignore Us: No effects, loser gets no action
    - Plane Shift: Shift row or column one space (toroidal)
  • Played cards placed in holes (or discarded if none available)

DAMAGE & WIN CONDITION ✓
  • Damage accumulates in player damage piles
  • Card values: Ace=1, 2-10=face, Queen=11, King=12
  • First to 25+ total damage loses immediately
  • Game ends automatically on defeat

ARCHITECTURE DESIGN
═══════════════════════════════════════════════════════════════════════════════

LAYER SEPARATION

  ┌─────────────────────────────────────────────────┐
  │ UI LAYER (pygame-ce)                            │
  │ window.py, board_renderer.py                    │
  │ - Rendering, input handling                     │
  │ - Communicates via Action objects               │
  └─────────────────────────────────────────────────┘
                        ▲
                        │ Actions
                        ▼
  ┌─────────────────────────────────────────────────┐
  │ ENGINE LAYER (Pure Python)                      │
  │ game_state.py, board.py, cards.py               │
  │ - Game logic, rules, state                      │
  │ - NO pygame imports (100% headless)             │
  │ - Testable without display                      │
  └─────────────────────────────────────────────────┘
                        ▲
                        │ State Queries
                        ▼
  ┌─────────────────────────────────────────────────┐
  │ AGENTS (Pluggable)                              │
  │ Human, Heuristic, RL, AI vs AI                  │
  │ - Implement: choose_action(state) -> Action     │
  │ - Query: state, legal_moves, position, damage   │
  └─────────────────────────────────────────────────┘

KEY DESIGN PRINCIPLES

  1. No Pygame in Engine
     - Engine is 100% headless
     - Can run without display
     - Can be tested without UI
     - Can be used with different UIs

  2. Action-Based Communication
     - All player intentions are Action objects
     - Easy to log, replay, analyze
     - Natural for RL environments
     - Supports AI agents

  3. State Query Interface
     - Agents query read-only game state
     - No direct state modification from UI
     - Clear separation of concerns
     - Easy to validate

  4. Type Safety
     - Full type hints throughout
     - Enables IDE autocomplete
     - Supports static type checking
     - Better documentation

  5. Testability
     - All core logic is unit testable
     - No pygame dependencies in tests
     - 7 comprehensive test cases
     - 100% pass rate
